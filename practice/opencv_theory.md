## 1. 디지털 이미지란?

### 이미지의 본질

디지털 이미지는 **픽셀(Pixel)의 2D 배열**입니다.
픽셀은 "Picture Element"의 줄임말로, 이미지를 구성하는 가장 작은 단위예요.

```
이미지 = numpy 배열 (H, W) 또는 (H, W, C)

H = 세로 픽셀 수 (Height)
W = 가로 픽셀 수 (Width)
C = 채널 수 (Channel) - 흑백: 1, 컬러: 3
```

### 픽셀값의 의미

```
흑백 이미지: 픽셀값 0~255
  0   = 완전 검정
  128 = 중간 회색
  255 = 완전 흰색

컬러 이미지: 픽셀값 (B, G, R) 각각 0~255
  (0,   0,   0  ) = 검정
  (255, 0,   0  ) = 파랑  ← OpenCV는 BGR 순서!
  (0,   255, 0  ) = 초록
  (0,   0,   255) = 빨강
  (255, 255, 255) = 흰색
```

### 이미지 해상도와 용량

```
해상도 = 가로 픽셀 수 × 세로 픽셀 수
예: 1920 × 1080 = 약 200만 픽셀 (Full HD)

용량 계산:
  흑백: H × W × 1 byte
  컬러: H × W × 3 byte
  예: 1920 × 1080 컬러 = 약 6MB (압축 전)
```

### ⚠️ OpenCV의 BGR vs 일반 RGB

```
일반 세상: RGB (Red, Green, Blue) 순서
OpenCV:   BGR (Blue, Green, Red) 순서  ← 역사적 이유로 다름!

이게 왜 문제냐면:
  cv2.imread()로 읽으면 BGR
  matplotlib.imshow()는 RGB 기대
  → 그냥 show하면 빨간 사진이 파랗게 나옴!

해결:
  img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)
```

## 2. 색상 공간 (Color Space)

### 색상 공간이란?

색을 수치로 표현하는 방법의 체계입니다.
같은 색을 어떤 기준으로 숫자로 나타내느냐에 따라 다양한 색상 공간이 존재해요.

---

### 2-1. RGB / BGR

가장 기본적인 색상 공간. 빛의 3원색으로 표현합니다.

```
R (Red)   : 0~255
G (Green) : 0~255
B (Blue)  : 0~255

장점: 직관적, 모니터 출력에 최적
단점: 조명 변화에 민감 → 같은 물체도 밝기에 따라 RGB값이 크게 달라짐
```

```
        R    G    B
빨강: [255,   0,   0]
초록: [  0, 255,   0]
파랑: [  0,   0, 255]
노랑: [255, 255,   0]  ← R+G 합치면 노랑
흰색: [255, 255, 255]
검정: [  0,   0,   0]
```

---

### 2-2. Grayscale (흑백)

컬러 → 흑백으로 변환할 때 단순히 평균 내는 게 아니라,
**인간 눈의 민감도**를 반영한 가중 평균을 씁니다.

```
Gray = 0.114×B + 0.587×G + 0.299×R

왜 G 비중이 높냐면?
→ 인간의 눈은 초록색에 가장 민감하기 때문!
→ G: 59%, R: 30%, B: 11% 비율로 인식
```

```
언제 쓰냐?
  - 색상 정보가 필요 없을 때 (엣지, 모양 분석)
  - 연산량 줄이기 (채널 1개라 3배 빠름)
  - 이진화, 엣지 검출의 전처리 단계
```

---

### 2-3. HSV ← 실무에서 매우 중요!

```
H (Hue)        : 색상 (0~179) - 색의 종류
S (Saturation) : 채도 (0~255) - 색의 진함
V (Value)      : 명도 (0~255) - 색의 밝기
```

```
색상환 (Hue 값):
  0   = 빨강
  30  = 주황
  60  = 노랑
  90  = 초록
  120 = 하늘
  150 = 파랑
  179 = 다시 빨강 (원형!)

※ OpenCV는 Hue를 0~360이 아닌 0~179로 표현 (절반으로 압축)
```

```
RGB vs HSV 비교:
  상황: 맑은 날 vs 흐린 날의 파란 하늘 사진

  RGB: 맑은날 (135, 206, 235), 흐린날 (100, 149, 237) → 값이 많이 달라짐
  HSV: 맑은날 H≈200, 흐린날 H≈210 → H값은 비슷하게 유지됨!

→ HSV는 조명 변화에 강해서 특정 색 추출할 때 훨씬 안정적!
→ 홍수 영역(파란 물) 추출할 때 HSV 마스킹이 효과적
```

**HSV 색상 범위 치트시트:**

| 색상 | H 범위 | S 범위 | V 범위 |
|------|--------|--------|--------|
| 빨강 (1) | 0~10 | 100~255 | 100~255 |
| 빨강 (2) | 160~179 | 100~255 | 100~255 |
| 주황 | 11~25 | 100~255 | 100~255 |
| 노랑 | 26~35 | 100~255 | 100~255 |
| 초록 | 36~85 | 100~255 | 100~255 |
| 파랑 | 100~130 | 100~255 | 100~255 |
| 하늘 (물) | 85~130 | 50~255 | 50~255 |
| 흰색 | 0~179 | 0~30 | 200~255 |
| 검정 | 0~179 | 0~255 | 0~50 |

---

### 2-4. LAB 색상 공간

```
L : 밝기 (Lightness, 0~100)
A : 초록~빨강 축 (-128~127)
B : 파랑~노랑 축 (-128~127)

특징:
  - 인간 시각과 가장 유사한 색상 공간
  - 조명 변화에 매우 강함
  - 두 색의 "거리" = 실제 눈으로 느끼는 차이와 일치
  - 딥러닝 전처리에서 가끔 사용
```

## 3. 이진화 (Thresholding)

### 이진화란?

이미지를 **흑(0)과 백(255) 두 값만으로** 단순화하는 작업입니다.
"이 픽셀이 관심 영역인가, 아닌가?" 를 판단하는 첫 단계예요.

```
입력: 흑백 이미지 (0~255 사이 다양한 값)
출력: 이진 이미지 (0 또는 255만 존재)

활용:
  - 물체와 배경 분리
  - 노이즈 제거
  - Segmentation 마스크 생성
  - 컨투어 추출의 전처리
```

---

### 3-1. 단순 이진화 (Simple Threshold)

```
규칙: 픽셀값 > T이면 255, 아니면 0
      (T = 사람이 직접 정하는 임계값)

문제점:
  - T값을 사람이 직접 설정해야 함
  - 조명이 바뀌면 T값도 바꿔야 함
  → 실무에서는 거의 안 씀
```

---

### 3-2. Otsu 이진화 ← 실무 표준

```
핵심 아이디어:
  "픽셀값의 분포를 보고, 가장 잘 나누는 T를 자동으로 찾자"

작동 원리:
  1. 이미지의 히스토그램(픽셀값 분포) 계산
  2. 모든 가능한 T값(0~254)에 대해 시도
  3. 각 T로 나눴을 때 두 그룹의 "분산"이 최소가 되는 T 선택
  4. 그 T로 이진화 수행

언제 잘 동작하냐?
  → 히스토그램이 두 개의 봉우리(bimodal)를 가질 때 최고
  → 밝은 배경 + 어두운 물체, 또는 그 반대
  → 홍수 이미지에서 물 영역과 육지 분리에 효과적

언제 잘 안 되냐?
  → 조명이 불균일할 때 (한쪽은 밝고 한쪽은 어두운 이미지)
  → 이럴 때는 Adaptive 사용
```

---

### 3-3. Adaptive 이진화

```
핵심 아이디어:
  "이미지 전체에 하나의 T를 쓰지 말고,
   각 픽셀의 주변 영역(neighborhood)마다 T를 따로 계산하자"

작동 원리:
  1. 각 픽셀 주변 N×N 영역 (blockSize) 선택
  2. 그 영역의 픽셀 평균 (또는 가우시안 가중 평균) 계산
  3. 평균 - C 를 그 픽셀의 임계값으로 사용
  4. 픽셀값 > 임계값이면 255, 아니면 0

파라미터:
  blockSize : 주변 영역 크기 (홀수, 보통 11~51)
  C         : 평균에서 뺄 상수 (보통 2~10)

언제 쓰냐?
  → CCTV처럼 역광, 그림자가 있는 영상
  → 야간 영상
  → 조명이 균일하지 않은 환경
```

## 4. 엣지 검출 (Edge Detection)

### 엣지란?

이미지에서 **픽셀값이 급격히 변하는 경계선**입니다.
물체의 윤곽, 그림자 경계, 질감 변화 등이 엣지로 나타나요.

```
수학적 의미:
  픽셀값의 변화율 = 미분값 (Gradient)
  변화율이 크다 = 엣지 가능성 높음

이미지는 2D이므로:
  X방향 미분 (Gx) = 가로 방향 변화
  Y방향 미분 (Gy) = 세로 방향 변화
  전체 엣지 강도  = √(Gx² + Gy²)
```

---

### 4-1. Sobel 필터

```
개념:
  X방향, Y방향 미분을 각각 3×3 커널로 근사화

Sobel X 커널 (세로 엣지 검출):     Sobel Y 커널 (가로 엣지 검출):
  [-1, 0, +1]                         [-1, -2, -1]
  [-2, 0, +2]                         [ 0,  0,  0]
  [-1, 0, +1]                         [+1, +2, +1]

작동 방식:
  각 픽셀에 커널을 "덮어씌워서" 주변값과 가중합 계산
  왼쪽-오른쪽 픽셀값 차이가 클수록 → Gx 큼 → 세로 엣지
  위-아래 픽셀값 차이가 클수록   → Gy 큼 → 가로 엣지

특징:
  - 방향 정보 포함 (어느 방향의 엣지인지 알 수 있음)
  - 노이즈에 약함
  - 엣지가 두껍게 나옴
```

---

### 4-2. Canny 엣지 ← 가장 많이 씀

```
Sobel의 단점(노이즈, 두꺼운 엣지)을 보완한 알고리즘

처리 단계 (4단계):

  1단계. 가우시안 블러로 노이즈 제거
         → 미분 전에 이미지를 살짝 흐리게 만들어 노이즈 줄임

  2단계. Sobel로 그라디언트(Gx, Gy) 계산
         → 엣지 강도와 방향 계산

  3단계. Non-Maximum Suppression (NMS)
         → 엣지 방향으로 주변 픽셀보다 값이 큰 픽셀만 남김
         → 두꺼운 엣지를 얇은 1픽셀 선으로 만듦

  4단계. Hysteresis Thresholding (이중 임계값)
         → 강한 엣지(high threshold 초과): 확실한 엣지로 확정
         → 약한 엣지(low~high 사이): 강한 엣지와 연결된 것만 유지
         → 매우 약한 엣지(low 미만): 제거

파라미터:
  threshold1 (low)  : 약한 엣지 하한값 (보통 50)
  threshold2 (high) : 강한 엣지 기준값 (보통 150)
  비율은 1:2 또는 1:3 권장

특징:
  - 노이즈에 강함
  - 얇고 깔끔한 엣지
  - 이중 임계값으로 끊김 최소화
  → CV 실무 표준 엣지 검출기
```

## 5. 마스킹 (Masking)

### 마스킹이란?

이미지에서 **관심 있는 영역(ROI, Region of Interest)만 선택**하고
나머지는 가리는(검정으로 만드는) 작업입니다.

```
마스크 = 같은 크기의 흑백 이미지
  흰색(255) = 이 픽셀을 사용
  검정(0)   = 이 픽셀을 버림

원리: 비트 AND 연산
  원본 픽셀값 AND 마스크 픽셀값
  = 원본 × (마스크/255)

  원본=150, 마스크=255 → 150 (유지)
  원본=150, 마스크=0   → 0   (제거)
```

---

### 5-1. 형태 기반 마스킹

```
특정 형태의 영역만 보고 싶을 때

사각형 ROI:
  roi = img[y1:y2, x1:x2]  ← 파이썬 슬라이싱으로 간단하게

원형/다각형 ROI:
  마스크 만들기 → cv2.circle() or cv2.fillPoly() 로 흰색 채우기
  → bitwise_and 적용

활용:
  - 카메라 화면 일부만 분석 (도로 영역만, 하천 영역만)
  - 리빙랩에서 특정 구역만 모니터링
```

---

### 5-2. 색상 기반 마스킹 (HSV) ← 실무 핵심

```
특정 색상만 추출하고 싶을 때

흐름:
  BGR 이미지 → HSV 변환 → 색상 범위 지정 → 마스크 생성 → 적용

cv2.inRange(hsv, lower, upper)
  → lower~upper 범위 안의 픽셀 = 255
  → 범위 밖의 픽셀 = 0

홍수 탐지 예시:
  물은 보통 파란색/회색 계열
  lower = [90, 50, 50]  (파란 계열 하한)
  upper = [130, 255, 255] (파란 계열 상한)

주의:
  빨간색은 Hue 0 근처 + 180 근처 두 군데에 걸쳐 있음
  → inRange 두 번 해서 bitwise_or로 합쳐야 함
```

---

### 5-3. Segmentation 마스크와의 관계

```
딥러닝 Segmentation 모델의 출력 = 마스크!

모델이 픽셀마다 "이게 물이냐 아니냐" 예측
→ 결과가 (H, W) 형태의 0/1 배열 = 마스크

이 마스크를 원본 이미지에 적용하면:
  물 영역만 빨간색으로 표시하기
  물 영역 면적 계산하기 (침수 범위 측정)
  물 영역 컨투어 추출하기 (경계선 시각화)

→ OpenCV 마스킹을 이해해야 딥러닝 결과를 활용할 수 있음!
```

## 6. 컨투어 (Contour)

### 컨투어란?

이진 이미지에서 **같은 색(흰색) 영역의 경계선 좌표**를 추출한 것입니다.
쉽게 말해 물체의 "윤곽선 좌표 목록"이에요.

```
입력: 이진 이미지 (흑백만 있는)
출력: 컨투어 리스트 (각 컨투어 = 경계 픽셀 좌표들의 배열)

contours = [
  array([[x1,y1], [x2,y2], ...]),  # 첫 번째 물체 윤곽
  array([[x1,y1], [x2,y2], ...]),  # 두 번째 물체 윤곽
  ...
]
```

---

### 6-1. 컨투어 검색 방법

```
cv2.RETR_EXTERNAL  : 가장 바깥 컨투어만 (실무에서 대부분 이것)
cv2.RETR_LIST      : 모든 컨투어 (계층 무시)
cv2.RETR_TREE      : 모든 컨투어 + 계층 구조 (구멍 포함)

cv2.CHAIN_APPROX_SIMPLE  : 꼭짓점만 저장 (메모리 절약, 보통 이것 씀)
cv2.CHAIN_APPROX_NONE    : 경계의 모든 픽셀 저장 (메모리 많이 씀)
```

---

### 6-2. 컨투어로 할 수 있는 것

```
면적 계산:
  area = cv2.contourArea(c)
  → 홍수 침수 면적(픽셀 단위) 계산 가능

둘레 계산:
  perimeter = cv2.arcLength(c, True)  # True = 닫힌 곡선

바운딩 박스:
  x, y, w, h = cv2.boundingRect(c)
  → Segmentation 결과를 Detection 형식으로 변환할 때 사용

중심점:
  M  = cv2.moments(c)
  cx = int(M['m10'] / M['m00'])
  cy = int(M['m01'] / M['m00'])
  → 물체의 무게중심 좌표

최소 외접원:
  (x, y), radius = cv2.minEnclosingCircle(c)

볼록 껍질 (Convex Hull):
  hull = cv2.convexHull(c)
  → 컨투어의 볼록한 외곽선
```

---

### 6-3. 면적 필터링의 중요성

```
딥러닝 모델이나 이진화 결과에는 항상 작은 노이즈가 있음
→ 면적이 작은 컨투어 = 대부분 노이즈

필터링:
  large = [c for c in contours if cv2.contourArea(c) > min_area]

min_area 설정 기준:
  - 이미지 크기에 따라 다름
  - 1920×1080 이미지에서 보통 500~2000 픽셀²
  - 탐지하려는 최소 물체 크기 기준으로 설정
```

## 7. 동영상 처리 원리

### 동영상이란?

```
동영상 = 이미지(프레임)를 빠르게 연속으로 보여주는 것

FPS (Frames Per Second) = 초당 프레임 수
  24 fps  : 영화
  30 fps  : 일반 TV, 유튜브
  60 fps  : 게임, 고속 촬영
  120 fps : 스포츠, 슬로우 모션

CCTV 보통 15~30 fps
드론 영상 보통 30~60 fps
```

---

### 7-1. 프레임 처리 파이프라인

```
실시간 CV 시스템의 기본 구조:

while 영상이 끝나지 않았으면:
    프레임 읽기  (cap.read())
        ↓
    전처리      (크기 조정, 색변환, 노이즈 제거)
        ↓
    모델 추론   (딥러닝 예측)
        ↓
    후처리      (마스크 정제, 컨투어 추출)
        ↓
    결과 출력   (화면 표시 or 저장 or DB 전송)
```

---

### 7-2. 실시간 처리의 핵심 - 속도

```
처리 속도 목표:
  30 fps 영상을 실시간으로 처리하려면
  → 1프레임 처리 시간 < 33ms (1000ms / 30fps)

속도를 높이는 방법:
  1. 입력 해상도 줄이기  (1920×1080 → 640×480)
  2. 가벼운 모델 선택   (YOLOv8n이 YOLOv8x보다 훨씬 빠름)
  3. GPU 사용          (CPU 대비 10~100배 빠름)
  4. 배치 처리          (여러 프레임 묶어서 한번에 추론)
  5. 스킵 프레임        (매 3번째 프레임만 분석)

실무 팁:
  CCTV 16채널을 동시 처리해야 한다면?
  → GPU 서버 + 경량 모델이 필수
```

### 7-3. 코덱과 포맷

```
영상 포맷:
  .mp4  : H.264 코덱, 가장 범용적
  .avi  : 오래된 포맷, 무압축도 가능
  .mkv  : 자막 등 다중 스트림 지원

RTSP (Real Time Streaming Protocol):

```

## 8. 모폴로지 연산 (Morphology)

### 모폴로지란?

**이진 마스크의 형태(모양)를 변형**하는 연산입니다.
수학적 형태학(Mathematical Morphology)에서 유래했어요.

```
핵심 개념:
  구조 요소 (Structuring Element, SE) = 커널
  → 보통 3×3 또는 5×5 정방형 or 원형

  SE를 이미지 위로 슬라이딩하면서 픽셀 변환
```

---

### 8-1. 팽창 (Dilation)

```
원리:
  구조 요소 범위 안에 흰색(255)이 하나라도 있으면 → 중심 픽셀을 흰색으로

효과:
  흰색 영역이 커짐 (팽창)
  검정 구멍이 메워짐
  얇은 선이 두꺼워짐
  끊어진 영역이 연결됨

활용:
  - 마스크 영역을 약간 크게 만들고 싶을 때
  - 끊어진 경계 연결할 때
```

---

### 8-2. 침식 (Erosion)

```
원리:
  구조 요소 범위 안이 모두 흰색(255)이어야 → 중심 픽셀을 흰색으로
  (하나라도 검정이면 중심도 검정)

효과:
  흰색 영역이 줄어듦 (침식)
  작은 흰색 점(노이즈) 제거
  얇은 선이 사라짐
  큰 영역의 경계가 안쪽으로 들어옴

활용:
  - 마스크의 노이즈 제거
  - 붙어있는 두 물체 분리
```

---

### 8-3. 열기 (Opening) ← 실무 핵심

```
= 침식 후 팽창

효과:
  작은 흰색 노이즈 제거 (침식으로 사라짐)
  큰 영역은 유지됨 (팽창으로 복원)

활용:
  딥러닝 모델 출력 마스크 정제
  → 모델이 "아마도 물일 것 같다"고 예측한 작은 점들 제거
  → 확실한 큰 홍수 영역만 남김
```

---

### 8-4. 닫기 (Closing) ← 실무 핵심

```
= 팽창 후 침식

효과:
  마스크 내부의 작은 구멍 메움 (팽창으로 메워짐)
  외곽 형태는 유지됨 (침식으로 복원)

활용:
  홍수 마스크에서 물 위 떠다니는 물체로 생긴 구멍 메우기
  → 홍수 영역이 연속적으로 이어지게 만들기
```

---

### 8-5. 모폴로지 연산 순서 결정법

```
마스크 품질 개선 표준 순서:

  ① OPEN  (iterations=2) → 작은 노이즈 제거
  ② CLOSE (iterations=2) → 구멍 메우기
  ③ 면적 필터링           → 남은 작은 컨투어 제거

iterations(반복 횟수):
  1 = 커널 한 번 적용 (약하게)
  2 = 두 번 적용 (더 강하게)
  → 노이즈가 많으면 iterations 늘리기
  → 물체가 작으면 iterations 줄이기 (물체 사라질 수 있음)
```

## 9. 필터와 컨볼루션

### 컨볼루션이란?

**필터(커널)를 이미지 위로 슬라이딩하면서 가중합을 계산**하는 연산입니다.
딥러닝 CNN(Convolutional Neural Network)의 핵심 연산이기도 해요!

```
컨볼루션 계산:
  커널 중심을 픽셀 위에 올림
  → 커널과 해당 픽셀 주변값을 원소별 곱셈
  → 모두 더한 값 = 출력 픽셀값

예시 (3×3 평균 필터):
  커널:  [1/9, 1/9, 1/9]
         [1/9, 1/9, 1/9]
         [1/9, 1/9, 1/9]
  → 주변 9개 픽셀 평균 = 블러(흐림) 효과
```

---

### 9-1. 주요 필터 종류

```
블러 필터 (Low-pass filter):
  평균 블러   : 단순 평균 → 속도 빠름
  가우시안 블러: 중심에 가중치 더 줌 → 자연스러운 블러
  미디언 블러 : 중앙값 사용 → 소금후추 노이즈에 강함

엣지 필터 (High-pass filter):
  Sobel   : 방향성 엣지
  Laplacian: 모든 방향 엣지 (2차 미분)

커스텀 필터:
  커널 값을 직접 만들어서 원하는 효과 적용 가능
```

---

### 9-2. 왜 딥러닝에서 중요한가?

```
CNN(Convolutional Neural Network)의 핵심:
  "필터(커널)의 값을 사람이 만들지 않고, 학습으로 자동으로 찾자"

전통 CV:
  사람이 직접 커널 설계 → 엣지 검출, 블러 등

딥러닝 CV:
  데이터로부터 최적 커널 자동 학습
  → 초반 레이어: 엣지, 선 검출 커널 학습
  → 중반 레이어: 모서리, 패턴 검출 커널 학습
  → 후반 레이어: 물체 전체 형태 인식 커널 학습

→ OpenCV 컨볼루션을 이해하면 CNN 구조 이해가 훨씬 쉬워짐!
```

## 10. CV 전체 파이프라인 흐름

### 전통 CV vs 딥러닝 CV

```
전통 CV (Rule-based):
  원본 이미지
      ↓
  전처리 (블러, 색변환)
      ↓
  특징 추출 (엣지, 색상 마스킹)
      ↓
  규칙 기반 판단 (임계값, 면적 비교)
      ↓
  결과

장점: 빠름, 설명 가능, 데이터 불필요
단점: 복잡한 환경에서 성능 한계, 사람이 규칙 다 만들어야 함
```

```
딥러닝 CV (Learning-based):
  원본 이미지
      ↓
  전처리 (크기 조정, 정규화)
      ↓
  CNN 모델 추론 (자동 특징 추출 + 판단)
      ↓
  후처리 (마스크 정제, 컨투어 추출)
      ↓
  결과

장점: 복잡한 환경도 잘 처리, 성능 높음
단점: 데이터 필요, 학습 시간 필요, 블랙박스
```

---

### 실무 파이프라인 전체 구조

```
[입력]
  CCTV / 드론 영상 / 위성 이미지
      ↓
[수집]
  VideoCapture (로컬) or RTSP (네트워크 카메라)
      ↓
[전처리]
  해상도 조정 → 색상 변환 → 정규화 (0~1)
      ↓
[추론]
  딥러닝 모델 (U-Net, YOLOv8 등)
      ↓
[후처리]
  이진화 → 모폴로지 → 컨투어 추출 → 면적 필터
      ↓
[결과 활용]
  시각화 / DB 저장 / 알림 발송 / 대시보드
```

---

### 성능 지표 (Segmentation)

```
Segmentation 모델의 성능을 측정하는 지표들:

IoU (Intersection over Union):
  예측 마스크와 정답 마스크가 얼마나 겹치는지
  IoU = 겹치는 영역 / 합친 영역
  1.0 = 완벽, 0.0 = 전혀 안 맞음
  실무 기준: 0.7 이상이면 괜찮음

mIoU (mean IoU):
  여러 클래스의 IoU 평균
  (배경, 물, 건물 등 클래스별 IoU를 평균)

Dice Score (F1 Score):
  Dice = 2 × |A∩B| / (|A| + |B|)
  IoU와 비슷하지만 작은 물체에 더 관대함
  의료/홍수 분야에서 자주 씀

Pixel Accuracy:
  전체 픽셀 중 올바르게 분류된 픽셀 비율
  단순하지만 클래스 불균형에 약함
  (물 10%, 배경 90%일 때 배경만 맞춰도 90% 나옴)
```
